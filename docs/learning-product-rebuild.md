# 星月英语学习产品与前后端重建方案

版本基线: 2.16.1

## 目标

星月英语的核心不是“功能越多越好”，而是把英语学习必需路径做成每天能执行的闭环:

1. 导入真实素材。
2. 生成可理解的双语字幕或段落。
3. 逐句精听和重播。
4. 用户手动收词。
5. 主动回忆练习。
6. FSRS 间隔复习。
7. TTS、词典、统计提供反馈。

Wordle、成就、看描述猜词只作为低压力巩固，不应阻塞今日主线。

## 学习科学分层

| 层级 | 用户需求 | 必需能力 | 证据 |
| --- | --- | --- | --- |
| 输入 | 有东西可学 | 本地/直链/平台导入、字幕或 ASR | 内容状态、字幕条数、失败原因 |
| 理解 | 听懂真实材料 | 固定播放器、双语时间线、单句重播 | 当前字幕、重播次数、精听记录 |
| 词汇 | 留住真正不会的词 | 点词查词、手动加入、来源句 | 生词上下文、保存时间、高亮 |
| 记忆 | 按遗忘曲线复习 | FSRS 四档反馈、到期优先 | 到期数、下次复习时间 |
| 输出 | 能主动回忆 | 打字背词、听写、逐句默写 | 练习会话、正确率、题数 |
| 支持 | 听发音和理解词义 | TTS 开关、缓存、词典补全 | 缓存命中、云端/系统兜底状态 |
| 反馈 | 知道下一步 | 今日路径、统计、学习压力 | 今日完成、听力分钟、正确率 |

## 前端重建

前端不再以“功能宫格”为中心，改成四个清晰分区:

- `今日路径`: 默认页，只显示用户今天应该做什么。
- `练习舱`: 打字、听写、逐句默写、错词回流；这是必做练习区。
- `词库阶梯`: 通用英语、IELTS、TOEFL 的阶段进度。
- `能力地图`: 解释能力来源和辅助入口，游戏/成就只放这里或练习舱辅助区。

播放页保持固定播放器和可滚动字幕，句子旁放重播、英文 TTS、中文 TTS、双语 TTS。字幕为空时必须显示原因和下一步。

## 后端/服务层重建

当前项目是 Android-first，没有独立服务器。这里的“后端”先重建为本地服务层，后续可平滑拆成远端服务。

| 层 | 当前落点 | 职责 |
| --- | --- | --- |
| Domain Core | `core/` | 学习架构、字幕状态机、学习路径、练习、FSRS、游戏规则 |
| Application Service | `androidApp/data/XingYueRepository.kt` | 组织导入、保存、复习、TTS 缓存、统计记录 |
| Local Backend | Room + cache | 内容、字幕、生词、attempt、TTS mp3、平台元数据 |
| Cloud Boundary | 百炼、yt-dlp | ASR、翻译、词典、TTS、公开平台解析 |
| UI Shell | Compose | 只展示状态和下一步，不直接决定学习逻辑 |

如果后续新增独立后端，优先拆出三类 API:

- `Import Job API`: 平台解析、字幕下载、ASR 队列。
- `Learning Sync API`: 内容、生词、attempt、FSRS 状态同步。
- `Cloud Proxy API`: 百炼 Key 托管、用量限额、失败重试。

本轮不新增远端服务，避免在平台解析和用户素材隐私上引入不可控依赖。

## 功能取舍规则

- 必需: 没有它就不能完成“导入素材 -> 听懂 -> 收词 -> 复习 -> 主动回忆”的能力。
- 支持: 能提高效率，但失败时必须有本地或系统兜底。
- 可选: 游戏、成就、视觉激励；不能成为完成今日路径的前置条件。

这套规则已在 `EnglishLearningArchitecture` 中固化，并有单元测试防止后续把游戏或统计误放进必需主线。
